# 画像アップロード、承認フロー、自動通知の実装まとめ

## 1. 画像付きタスク完了機能（既存実装）

### 流れ

- **受信BOX**でタスクを「受け入れる」と、そのタスクは **タスク一覧** と **ダッシュボード** に表示されます。
- **タスク一覧** または **ダッシュボード** で、届いたタスク（他者から送られたタスク）の **「完了」** を押すと、**カメラで撮影** または **フォトライブラリから選択** のダイアログが出ます。
- 画像を選ぶと **Firebase Storage** の `task_completion_images/` にアップロードされ、アップロード後の URL が Firestore のタスクの **completion_image_url** に保存されます。
- 同時にタスクの **status** が **pending_approval** に更新されます。

### 関連コード

- `TaskCardView`: 写真証拠タスクで「完了」タップ → カメラ/フォトライブラリ選択 → 画像を `onComplete(task, photo)` で渡す。
- `TaskListView` / `DashboardView`: `onComplete` 内で `StorageService.uploadCompletionImage` → `taskViewModel.completeTask(..., completionImageURL: url)`。
- `TaskRepository.reportTaskCompletion`: Firestore の `status: "pending_approval"`, `completion_image_url` を更新。

---

## 2. 承認フロー（既存実装）

### 流れ

- **送信者** は **受信BOX** または **設定** の **「承認待ち」** から、ステータスが **pending_approval** のタスク一覧（ApprovalPendingView）を開けます。
- 一覧では **相手が送った画像**（completion_image_url）を **AsyncImage** で表示し、読み込み中・失敗時はプレースホルダーを表示します。
- **「承認」** を押すと、タスクの status が **completed** になり、紐づく **ギフト** の **is_unlocked** が **true** に更新されます。
- **「差し戻し」** を押すと、タスクの status が **active** に戻ります。

### 関連コード

- `ApprovalPendingView` / `ApprovalPendingCardView`: 画像表示、承認/差し戻しボタン。
- `TaskViewModel.approveTaskCompletion` → `TaskRepository.approveTaskCompletion`: タスクを completed、ギフトを is_unlocked: true に更新。
- `TaskViewModel.rejectTaskCompletion` → `TaskRepository.rejectTaskCompletion`: タスクを active に戻す。

---

## 3. アクションに伴う自動プッシュ通知（新規：Cloud Functions）

Firestore の `tasks` 更新をトリガーに、**Cloud Functions** から FCM でプッシュを送信します。  
送信先は Firestore の **users** コレクションに保存されている **fcm_token** です。

### 3-1. タスク完了（画像アップロード）時 → 送信者へ

- **トリガー**: タスクの **status** が **pending_approval** に変わったとき。
- **送信先**: タスクの **sender_id** に紐づく **users/{sender_id}.fcm_token**。
- **文言**: 「〇〇さんがタスクを完了しました！承認してください」（〇〇は受信者の display_name）。

### 3-2. 送信者が承認した時 → 受信者へ

- **トリガー**: タスクの **status** が **completed** に変わったとき。
- **送信先**: タスクの **receiver_id** に紐づく **users/{receiver_id}.fcm_token**。
- **文言**: 「おめでとう！タスクが承認され、ギフトが解禁されました！」

### デプロイ方法

`functions/README.md` を参照してください。要約すると:

1. `cd functions && npm install`
2. `firebase use <プロジェクトID>`
3. `firebase deploy --only functions`

※ Firebase Console に APNs 認証キー（.p8）を登録済みである必要があります。
