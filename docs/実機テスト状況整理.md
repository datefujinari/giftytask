# 実機テスト状況整理

## ログから分かる現在の状況

### ✅ 問題なく動いているもの

| 項目 | ログ | 意味 |
|------|------|------|
| Firestore 読み込み | `[AuthManager] Firestore からデータを読み込み` / `displayName: たつや` | 認証・プロフィール取得は成功。ユーザー「たつや」でログイン済み。 |
| 通知許可 | `[NotificationService] authorizationStatus = 2` | **2 = 許可（authorized）**。ローカル通知は許可されている。 |

→ **ローカル通知（「テスト通知を送る」など）は、現状の設定で動作する状態です。**

---

### ❌ まだ解消していないもの（プッシュ通知まわり）

| 項目 | ログ | 意味 |
|------|------|------|
| APNs エンタイトルメント | `FCM: APNs registration failed: アプリケーションの有効な"aps-environment"エンタイトルメント文字列が見つかりません` | アプリに **Push Notifications** の capability が付いておらず、APNs に登録できていない。 |
| FCM トークン | `APNS device token not set before retrieving FCM Token` / `Declining request for FCM Token` / `refreshFCMTokenIfNeeded error: No APNS token specified` | 上記の結果、APNs トークンが無いため FCM トークンも取得できていない。 |

→ **リモートプッシュ（FCM）を使うには、Xcode で Push Notifications を有効にする必要があります。**

---

### ⚠️ 参考情報（挙動には直結しない）

| 項目 | ログ | 補足 |
|------|------|------|
| App Delegate の型 | `App Delegate does not conform to UIApplicationDelegate protocol`（GoogleUtilities） | SwiftUI の `@UIApplicationDelegateAdaptor` 利用時によく出る警告。実際には `AppDelegate` は `UIApplicationDelegate` に準拠しており、多くの場合は無視してよい。 |
| Method Swizzling | `FIRMessaging Remote Notifications proxy enabled, will swizzle...` | FCM がデフォルトでリモート通知のハンドラをすり替える旨の通知。手動統合にしたい場合のみ Info.plist で `FirebaseAppDelegateProxyEnabled = NO` にする。 |

---

## まとめ（現状の整理）

- **認証・Firestore・通知許可**  
  → 実機でも正常。ユーザー「たつや」でログインできており、通知は「許可」状態（authorizationStatus = 2）。
- **ローカル通知**  
  → 許可済みなので、「テスト通知を送る」などは動作する想定。
- **リモートプッシュ（FCM）**  
  → まだ APNs 登録が失敗しているため動いていない。  
  → **Xcode でターゲットに「Push Notifications」capability を追加**すれば、APNs トークン → FCM トークンが取得できるようになる。

次の対応候補：
1. **FCM を使う場合**  
   Signing & Capabilities で **Push Notifications** を追加し、実機で再ビルド・再実行。
2. **ローカル通知だけ確認する場合**  
   現状のまま「テスト通知を送る」で 5 秒後の通知が届くか確認する。
