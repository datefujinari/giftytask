# プッシュ通知（FCM）を有効にする — やること一覧

Xcode で **Push Notifications** と **Background Modes > Remote notifications** は追加済みとします。

---

## 1. Xcode でやること

### 1-1. クリーンビルドしてから実機で実行

1. メニュー **Product → Clean Build Folder**（Shift + ⌘ + K）
2. 実機「伊達 iPhone」を選んで **Run**（⌘ + R）

→ 新しいエンタイトルメントがプロビジョニングに反映されます。

### 1-2. 署名が「自動」か確認

- **Signing & Capabilities** で **Signing** が **Automatically manage signing** になっていると、Xcode がプロビジョニングを整えてくれます。
- **手動署名**の場合は、次の「2. Apple Developer 側」も必要です。

---

## 2. Apple Developer 側（手動署名の場合だけ）

**Automatically manage signing** にしていない場合のみ行います。

1. [Apple Developer](https://developer.apple.com/account) にログイン
2. **Certificates, Identifiers & Profiles → Identifiers** で **GiftyTask** の App ID を開く
3. **Capabilities** で **Push Notifications** にチェックが入っているか確認（入れて保存）
4. **Profiles** で、その App ID 用の **Development**（または Distribution）プロビジョニングプロファイルを **編集して再生成**し、Xcode にダウンロードし直す  
   （Xcode で「Download Manual Profiles」でも可）

---

## 3. 実機での確認手順

1. 実機でアプリを起動する（通知許可は「許可」のまま）
2. Xcode の **コンソール**で次を確認：

**成功している場合の例**

- `FCM: APNs registration failed...` が **出なくなる**
- `FCM token: xxxxx...` のようなログが出る
- `refreshFCMTokenIfNeeded error...` が **出なくなる**

**まだ失敗している場合**

- 同じく `APNs registration failed` / `aps-environment が見つかりません` が出る  
  → 1 のクリーンビルド＋実機 Run を再度行う、または 2 のプロビジョニングを見直す

---

## 4. まとめ（やる順）

| 順番 | やること |
|------|----------|
| 1 | Xcode で **Clean Build Folder** → 実機で **Run** |
| 2 | コンソールで **APNs registration failed** が消え、**FCM token** が出るか確認 |
| 3 | （手動署名だけ）Apple Developer で App ID の Push Notifications 有効化とプロファイル再生成 |

ここまでできれば、FCM トークンが取得され、Firestore の `fcm_token` も更新されるようになります。
