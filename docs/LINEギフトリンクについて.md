# LINEギフトへのリンクについて

## 実装

- **飛び先**: アンロック済みギフトの「利用する」タップで **LINEギフト公式トップ** `https://linegift.line.me/` を開く実装にしています。
- **設定箇所**:
  - モック: `GiftyTask/Utilities/PreviewContainer.swift` の `lineGiftOfficialURL` およびアンロック済みギフトの `giftURL`
  - 表示・遷移: `GiftyTask/Views/Gift/GiftCardView.swift` の `Link(destination: URL(string: giftURL)!)`
  - モデル: `GiftyTask/Models/Gift.swift` の `giftURL`（本番ではAPI等で取得したURLを設定）

## 法的な位置づけ（注意：法務確認は推奨）

- **LINEギフト利用規約**（https://terms2.line.me/linegift?lang=ja）を確認した範囲では、**「第三者のアプリからLINEギフトへのリンクを禁止する」旨の条文はありません**。
- 禁止されているのは主に次のような行為です（規約第5条）:
  - **5.1.22**: 当社の承諾を得ない**営利活動・宣伝広告**に本サービスを**利用する**行為（本サービス**内**での商品購入は除く）
  - **5.1.19**: 本サービス**外**で取引を進展させようと**持ちかける**行為 など

- **自アプリから「LINEギフトを開く」リンクを張る行為**は、
  - ユーザーをLINEギフトの公式ページへ送っているだけであり、
  - LINEギフト上で広告や営利活動を行うわけではなく、
  - 「サービス外で取引を持ちかける」の典型例（LINE外で代金を取る等）にも当たりにくい  
  ため、**一般的な「リンクを貼って遷移させる」使い方であれば、規約上問題になりにくい**と考えられます。

- 一方で、以下に該当する場合は規約・法務の確認を強く推奨します:
  - LINEギフトへの遷移に対して**アフィリエイト報酬**を受け取る
  - 「LINE公認」「LINEと提携」などと**誤解を招く表示**をする
  - LINEの**API・SDKを無許諾で利用**する（現在は公式URLへのリンクのみでAPIは未使用）

**結論**: 「利用する」で公式URL（`https://linegift.line.me/`）を開くだけの現状の実装は、規約上問題になりにくいと考えられますが、**商用利用・収益化・API連携を行う場合は、LINEヤフー株式会社の利用規約・開発者向けガイドおよび必要に応じて法務確認を行うことを推奨します。**
