# Cloud Functions デプロイ後の確認と次のステップ

## デプロイ内容の確認（問題なし）

デプロイされた **onTaskUpdated** は次の動きをします。

| トリガー | 条件 | 送信先 | 通知文言 |
|----------|------|--------|----------|
| Firestore `tasks/{taskId}` の更新 | status が **pending_approval** に変わった | 送信者（sender_id）の `users/{uid}.fcm_token` | 「〇〇さんがタスクを完了しました！承認してください」 |
| 同上 | status が **completed** に変わった | 受信者（receiver_id）の `users/{uid}.fcm_token` | 「おめでとう！タスクが承認され、ギフトが解禁されました！」 |

- **リージョン**: us-central1
- **ランタイム**: Node.js 22（2nd Gen）
- **クリーンアップ**: コンテナイメージは 30 日後に自動削除

実装と仕様に問題はありません。

## 次のやること

### 1. 動作確認（実機でフローを試す）

1. **端末 A（送信者）**: 相手の UID を入力してタスクを送信する。
2. **端末 B（受信者）**: 受信BOXでタスクを「受け入れる」→ タスク一覧でそのタスクを **画像付きで完了** する。
3. **端末 A**: 数秒以内に **「〇〇さんがタスクを完了しました！承認してください」** のプッシュが届くか確認する。
4. **端末 A**: 受信BOX（または設定）の「承認待ち」から該当タスクを開き、**「承認」** を押す。
5. **端末 B**: **「おめでとう！タスクが承認され、ギフトが解禁されました！」** のプッシュが届くか確認する。

※ 両方の端末で **通知を許可** し、Firestore の `users` に **fcm_token** が保存されている必要があります。

### 2. 届かない場合の確認

- Firebase Console → **Functions** → **onTaskUpdated** → **ログ** でエラーや「no fcm_token」が出ていないか確認する。
- 送信者・受信者それぞれの `users/{uid}` に **fcm_token** が入っているか、Firestore で確認する。
- アプリを一度終了して起動し直し、FCM トークンが再取得・保存されているか確認する。

### 3. （任意）firebase-functions のアップグレード

デプロイ時の警告「package.json indicates an outdated version of firebase-functions」を消したい場合:

```bash
cd /Users/itoutatsuya/kaihatu/taskapp/functions
npm install --save firebase-functions@latest
```

その後、問題なければ `npx firebase-tools deploy --only functions` で再デプロイできます。既存の動きを変えずに警告だけ解消する目的なら、後回しでも構いません。
